## 1. 개요

본 프로젝트는 **제8회 교육 공공데이터 AI 활용대회 공모 요건**에 따라 교육 분야 공공데이터를 필수적으로 활용합니다.

### 🎯 활용 공공데이터 개요
- **데이터 개수**: 2개 이상 주요 API 연동
- **데이터 범위**: 전국 17개 시도교육청 모든 학교
- **연동 방식**: 실시간 REST API (JSON)
- **활용 목적**: 전국 학교급식 통합 분석 및 시각화

## 2. 주요 공공데이터

### 🏛️ 2.1 NEIS 교육정보 개방 포털

**기관**: 교육부 · 17개 시도교육청  
**제공 서비스**: NEIS Open API  
**URL**: [https://open.neis.go.kr](https://open.neis.go.kr)

#### 활용 API
| API | 용도 | 레코드 수 |
|-----|------|---------|
| **학교기본정보** (`/schoolInfo`) | 전국 학교 기본 정보 | 11,000+ |
| **급식식단정보** (`/mealServiceDietInfo`) | 일일 급식 메뉴 | 일별 |

#### 핵심 필드
```
학교기본정보:
  - SD_SCHUL_CODE          (학교 고유 코드)
  - SCHUL_NM               (학교명)
  - SCHUL_KND_SC_NM        (학교 종류: 초등/중/고)
  - ATPT_OFCDC_SC_CODE     (시도교육청 코드)
  - ORG_RDNMA              (도로명주소)
  - FOND_YMD               (설립일)
  - LAT_CRDNT, LOT_CRDNT   (경위도 좌표)

급식식단정보:
  - MLSV_YMD               (급식 날짜)
  - MMEAL_SC_NM            (조식/중식/석식)
  - DDISH_NM               (식단 메뉴)
  - CAL_INFO               (칼로리)
  - NTR_INFO               (영양소 정보)
  - ORPLC_INFO             (원산지 정보)
```

### 🏫 2.2 학교알리미 공시정보 서비스

**기관**: 교육부 · 한국교육학술정보원(KERIS)  
**URL**: [https://open.schoolinfo.go.kr](https://open.schoolinfo.go.kr)

#### 활용 API
| API | 용도 |
|-----|------|
| **학교정보공시 API** | 학교별 평가/재정/교원 정보 |

#### 핵심 필드
```
- 학교운영계획

- 교직원 현황
- 급식 만족도 평가
```

## 3. API 상세 명세

### 📡 3.1 NEIS 학교기본정보 API

#### Request
```http
GET https://open.neis.go.kr/hub/schoolInfo
```

| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `KEY` | string | ✓ | 인증키 |
| `Type` | string | ✓ | `json` 또는 `xml` |
| `pIndex` | number | | 페이지 번호 (기본 1) |
| `pSize` | number | | 페이지 크기 (기본 100, 최대 1000) |
| `ATPT_OFCDC_SC_CODE` | string | | 시도교육청 코드 |

#### 시도교육청 코드표
```
B10: 서울특별시교육청    C10: 부산광역시교육청
D10: 대구광역시교육청    E10: 인천광역시교육청
F10: 광주광역시교육청    G10: 대전광역시교육청
H10: 울산광역시교육청    I10: 세종특별자치시교육청
J10: 경기도교육청         K10: 강원특별자치도교육청
M10: 충청북도교육청       N10: 충청남도교육청
P10: 전북특별자치도교육청 Q10: 전라남도교육청
R10: 경상북도교육청       S10: 경상남도교육청
T10: 제주특별자치도교육청
```

#### Response 예시
```json
{
  "schoolInfo": [
    {
      "head": [
        { "list_total_count": 866 },
        { "RESULT": { "CODE": "INFO-000", "MESSAGE": "정상 처리" } }
      ]
    },
    {
      "row": [
        {
          "SD_SCHUL_CODE": "7010001",
          "SCHUL_NM": "강남초등학교",
          "SCHUL_KND_SC_NM": "초등학교",
          "ATPT_OFCDC_SC_CODE": "B10",
          "ORG_RDNMA": "서울특별시 강남구 ...",
          "FOND_YMD": "19580301"
        }
      ]
    }
  ]
}
```

## 4. 데이터 흐름 및 활용

### 🔄 4.1 데이터 라이프사이클

```
[교육부 · 17개 시도교육청]
         │
         ▼
[NEIS/학교알리미 시스템]
         │ REST API (Open)
         ▼
[우리 대시보드 (클라이언트)]
         │
         ├─ 24시간 localStorage 캐싱
         │
         ▼
[데이터 파싱 & 변환]
         │
         ├─ 학교 정보 정규화
         ├─ 급식 메뉴 파싱
         ├─ 영양 정보 추출
         └─ 원산지 정보 구조화
         │
         ▼
[AI 분석 엔진]
         │
         ├─ Rule-based 평가 (7지표)
         ├─ Shannon Entropy 다양성
         ├─ 알레르기 탐지
         └─ Claude AI 챗봇
         │
         ▼
[시각화 레이어]
         │
         ├─ Leaflet 지도 (학교 위치)
         ├─ Chart.js (11종 차트)
         └─ 커스텀 SVG
```

### 🎯 4.2 실제 활용 사례

#### 사례 1: 학부모가 자녀 학교 급식 확인
```
1. "우리 아이 학교 찾기" 버튼 클릭
   → NEIS 학교기본정보 API 쿼리
2. 학교 선택
   → NEIS 급식식단정보 API (최근 7일)
3. AI 분석 엔진이 7지표 평가
   → 점수 + 해설 자동 생성
4. 카드뉴스 SNS 공유
```

#### 사례 2: 교사가 전국 벤치마킹
```
1. 지도 탭 → 시도교육청 "전국" 선택
   → 17개 교육청 API 병렬 호출 (Promise.all)
2. 차트 탭 자동 전환
   → 전국 전용 3개 차트 활성화
3. 순위 탭에서 전국 상위 학교 확인
   → Rule-based Score 기반
4. 우수 사례 벤치마킹
```
